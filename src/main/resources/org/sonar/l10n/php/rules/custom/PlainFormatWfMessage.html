Using MediaWiki's internationalization functions with the plain output format can have unexpected consequences, because this format does not parse language-specific magic words such as GENDER or PLURAL, and does not escape content for HTML. text format should be used instead, or escaped if the content should be sanitized.

Noncompliant Code Example

	function foo() {
		return wfMessage( 'foo' )->plain();
	}

Compliant Solution

	function foo() {
		return wfMessage( 'foo' )->text();
	}
